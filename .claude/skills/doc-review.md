# Documentation Review Checklist

Before committing documentation changes, run through this systematic review to catch common issues.

## Terminology Consistency

- [ ] No conflicting uses of the same term
- [ ] Product-specific terms capitalized correctly
- [ ] Generic terms don't conflict with product features
- [ ] Links use consistent terminology

## Section Structure

- [ ] Section titles follow established conventions
- [ ] "Common Use Cases" use bullet format with bold scenario names
- [ ] "Next Steps" sections link to related pages
- [ ] Heading hierarchy is correct (no skipped levels)

## Content Quality

- [ ] No duplicate content between parent and child pages
- [ ] Examples are accurate and up-to-date
- [ ] Screenshots referenced in text actually exist
- [ ] File paths in examples are correct

## Technical Accuracy

- [ ] Product feature names are correct
- [ ] CLI commands and code examples work
- [ ] Links point to existing pages
- [ ] Cross-references are accurate

## Formatting

- [ ] Markdown formatting is consistent
- [ ] Code blocks have language tags
- [ ] Lists use consistent bullet style
- [ ] Bold/italic usage follows conventions

## Git Hygiene

- [ ] Commit message explains what and why
- [ ] Related changes grouped in one commit
- [ ] No unintended files included
- [ ] Diff reviewed for unintended changes

Run this checklist before committing significant documentation changes.
